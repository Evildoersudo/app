# 七、命令状态机可视化结构

```
点击关闭
   ↓
按钮 loading
   ↓
发送 MQTT CMD
   ↓
等待 ACK (3秒)
   ↓
成功 → 更新 UI
   ↓
失败 → Toast 超时
```

插孔卡片增加：

```
状态条：
ON → OFF
loading → 变灰
```

# 八、视觉层级建议

层级分为：

1️⃣ 强卡片（今日用电）
2️⃣ 状态卡（当前功率）
3️⃣ 操作卡（控制）
4️⃣ 信息卡（事件）

避免全部灰色。

---

# 九、页面优先级（真实用户路径）

学生打开 App 流程：

```
首页 → 看是否安全
   ↓
点击插排 → 控制设备
   ↓
异常 → 看告警
   ↓
偶尔 → 看统计
```

所以：

首页必须最“爽”。
