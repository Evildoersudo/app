# 三、插排页面优化（重点）

当前问题：

* 信息有点拥挤
* ON/OFF 太普通
* 缺少状态反馈
* 控制不够“爽”

---

## 优化建议

### ① 插孔卡片改为“强状态卡”

现在：

```
插孔1
功率 80W
设备 PC
关闭按钮
```

建议改为：

```
[ 插孔1 ]
🟢 ON
80W
PC
```

按钮改成：

* 开 → 蓝色
* 关 → 灰色
* 发送中 → loading 状态

---

### ② 操作反馈必须加强

点击关闭时：

* 按钮变 loading
* 3秒内没 ack → toast 超时
* ack 成功 → 成功提示

否则学生会怀疑没反应。

---

### ③ 批量操作优化

现在“全部开启 / 全部关闭”：

建议：

* 弹出确认框
* 显示“逐个发送命令”

增加：

```
正在执行 3/4
```

会很专业。

---

# 四、告警页面优化

现在：

```
CMD_TIMEOUT 命令
cmd_xxx 超时
```

对普通学生太工程化。

---

## 优化建议

把告警翻译成人话：

```
⚠ 插孔3 控制失败
请检查网络或设备电源
```

不要出现：

* cmd_id
* CMD_TIMEOUT

可以在详情里展开。

---

### 增加筛选功能

* 今日
* 本周
* 未处理
